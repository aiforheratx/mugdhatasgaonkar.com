Native · macOS

AgentPulse & usagetracker

Two small Swift menu-bar apps. Neither is ambitious. Both get used every single day, which is more than I can say for some things I've spent months on.

AgentPulse

The desktop face of Mbrain's registry: every agent and whether it's actually alive, sitting in the menu bar. It answers "is the fleet up?" in a glance instead of a terminal session.

A dashboard you have to deliberately open is a dashboard you check when you're already worried. Putting it in the menu bar means noticing a dead agent on the way to doing something else — which is when you'd actually want to know.

usagetracker

Real-time AI usage against the rolling 5-hour and 7-day windows. Built because hitting a limit mid-task is disruptive in a way that's entirely preventable if you can see it coming.

Why native, for something this small

Both could have been scripts. Neither would have been used. The menu bar is the only surface that's always visible without being in the way, and reaching it means writing actual Swift rather than wrapping a web view.

These are the smallest things on this site and they make a case I believe in: the tool that's always in front of you beats the better tool you have to go and open. Ambition is not the same as usefulness.
