AgentPulse & usagetracker

Two small Swift menu-bar apps. Neither is ambitious. Both get used every single day, which is more than I can say for some things I've spent months on.

AgentPulse

The desktop face of Mbrain's registry: every agent and whether it's actually alive, sitting in the menu bar. A dashboard you have to deliberately open is a dashboard you check when you're already worried. Putting it in the menu bar means noticing a dead agent on the way to doing something else — which is when you'd actually want to know.

usagetracker

Real-time AI usage against the rolling 5-hour and 7-day windows. Built because hitting a limit mid-task is disruptive in a way that's entirely preventable if you can see it coming.

How they work

  1. Poll

    AgentPulse reads Mbrain's registry and probes the same ports the fleet answers on — live truth, not cached status.

    AgentPulse

  2. Meter

    usagetracker reads AI usage and holds it against the rolling 5-hour and 7-day limits.

    usagetracker

  3. Surface

    Both render into the one surface that's always visible without being in the way: the macOS menu bar.

    Menu bar

  4. Warn early

    Amber before the cap bites, red when something's down — the whole point is knowing before it costs you.

How Claude is used

  • built_withBoth apps were built with Claude Code — real Swift and AppKit, not a wrapped web view, because reaching the menu bar properly demands it.
  • watching_itselfusagetracker exists to watch Claude usage — the tool watches the tool, so a coding session never dies mid-thought at a rate limit.
  • fleet_glueAgentPulse is the menu-bar face of an agent fleet that's operated through Claude — the same registry Claude reads is the one the app renders.

What is used

  • LANGUAGESwift — native macOS, AppKit menu-bar apps
  • DATAMbrain's agents.json registry · live port probes · usage telemetry
  • SURFACEmacOS menu bar — always visible, never in the way

Why native, for something this small

Both could have been scripts. Neither would have been used. These are the smallest things on this site and they make a case I believe in: the tool that's always in front of you beats the better tool you have to go and open. Ambition is not the same as usefulness.
