CommCoach

A communication coach in two halves: it turns the documents you're supposed to have read into audiobooks, and it listens back while you practise saying things out loud.

10day personalized executive coaching program
100%of practice audio stays on the machine

1sentence to Siri starts a session

The problem

Two things I could never make time for. The reading pile — papers and books that stay unread because sitting down with them competes with everything else. And spoken delivery, which is the one skill that only improves by doing it badly first, ideally where nobody is watching.

How a session flows

  1. Drop

    A PDF lands in an iCloud folder. A FastAPI service is watching it.

    iCloud + FastAPI

  2. Listen or speak

    Documents become audiobooks for the walk; or you record a practice run-through in the iOS app — one Siri sentence to start.

    iOS app

  3. Transcribe

    Whisper turns the recording into text — locally. Unflattering practice audio never earns a network trip.

    Whisper · local

  4. Coach

    The coaching engine scores pace, filler words, and clarity, and Claude builds the personalized 10-day program and meeting analyses.

    Claude + local LLM

  5. Track

    Progress lands in SQLite and a Streamlit dashboard shows the trend — the proof that practice is working.

    SQLite · Streamlit

How Claude is used

  • the_planClaude designs the personalized 10-day executive communication program and analyses meetings — the strategic layer, working from text.
  • the_audioPractice recordings never reach Claude or any cloud: Whisper transcribes and an Ollama-hosted local model gives delivery feedback, on-machine.
  • the_splitThe design rule: cloud intelligence for what's already public (your goals, your calendar), local intelligence for what's private (your voice, your stumbles).

What is used

  • CLIENTiOS app with Siri integration — one sentence to start
  • TRACKINGSQLite progress store · Streamlit + Plotly dashboard

Why local

Practice recordings are unflattering by design — that's the point of them. Shipping that audio to someone else's server so it can be transcribed and possibly retained was never a trade I wanted to make. Running the models locally removed the question entirely, and as a side effect made the thing free to use as often as I like.

There's a broader lesson I keep relearning: the cost model shapes the behaviour. When every run costs a fraction of a cent, you ration yourself without noticing. When it costs nothing, you actually practise.
